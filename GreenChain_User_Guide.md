# GreenChain 系统使用指南

## 1. 系统概述

GreenChain 是一个综合供应链可持续性平台，帮助企业跟踪、分析和优化其环境影响。系统提供全面的工具来监控碳排放量、管理供应商、跟踪运输路线，并生成详细的可持续性报告。

### 系统主要功能：
- 核心可持续性指标监控
- 碳排放趋势分析与可视化
- 供应商管理与认证状态跟踪
- 全球供应链地图可视化
- 运输跟踪与管理
- 可持续性报告生成

## 2. 用户类型及使用指南

GreenChain 系统支持四种用户角色，每种角色都有特定的权限和功能访问权限：

### 2.1 管理员 (ADMIN)

**角色概述**：管理员拥有系统的最高权限，可以管理所有功能和用户。

**主要功能**：
- 管理所有用户账户
- 管理所有供应商信息
- 查看和分析所有碳排放数据
- 生成全系统报告
- 配置系统设置

**使用流程**：
1. **登录系统**：访问登录页面，输入管理员凭据
2. **系统概览**：登录后进入首页，查看系统主要功能模块
3. **供应商管理**：在供应商管理页面添加、编辑或删除供应商信息
4. **数据分析**：在核心指标页面查看系统的关键可持续性指标
5. **报告生成**：使用报告生成功能创建详细的可持续性报告

**建议截图位置**：
- 登录页面：`/login` - 展示管理员登录界面
- 供应商管理页面：`/supplier-management` - 展示管理员如何管理供应商
- 核心指标页面：`/core-indicators` - 展示系统的关键可持续性指标

### 2.2 可持续发展经理 (SUSTAINABILITY_MANAGER)

**角色概述**：可持续发展经理负责监控和分析供应链的环境影响，制定可持续发展策略。

**主要功能**：
- 查看和分析碳排放数据

**使用流程**：
1. **登录系统**：访问登录页面，输入可持续发展经理凭据
2. **数据分析**：在核心指标页面查看关键可持续性指标和历史分析
3. **供应商管理**：管理供应商信息，重点关注环境认证状态
4. **运输跟踪**：在运输跟踪页面监控运输活动及其环境影响

- **优化查询**：合理使用系统的查询功能，避免过度查询
- **使用批量操作**：在处理多个项目时使用批量操作功能

- **使用强密码**：为所有用户账户设置强密码
- **定期更新密码**：定期更改用户密码，提高账户安全性
- **限制访问权限**：根据用户角色设置适当的访问权限
- **保护敏感数据**：避免在公共场合展示敏感的环境数据

### 6.1 常见问题

| 无法登录系统 | 用户名或密码错误 | 检查用户名和密码，确保正确输入 |

## 7. 测试方法

### 7.1 测试方法概述

GreenChain 平台采用**混合测试方法**，结合自动化测试和手动测试，确保系统的功能完整性、安全性和用户体验。测试方法根据后端和前端的不同技术栈进行定制，以最大化测试效率和覆盖率。

### 7.2 后端测试方法

#### 7.2.1 单元测试（Unit Testing）

**测试框架**：JUnit 5（Spring Boot Test 默认包含）

**测试范围**：
- 单个方法的逻辑验证
- 数据模型验证
- 业务规则验证
- 边界条件测试

**测试示例**：
```java
@Test
void testCalculateCarbonEmission() {
    Shipment shipment = new Shipment();
    shipment.setDistanceKm(100.0);
    shipment.setCargoWeightTons(10.0);
    shipment.setTransportMode(new TransportMode("TRUCK", 0.2));
    
    double emission = carbonEmissionService.calculateEmission(shipment);
    assertEquals(200.0, emission, 0.01);
}
```

**测试执行**：
```bash
mvn test
```

#### 7.2.2 集成测试（Integration Testing）

**测试框架**：Spring Boot Test

**测试范围**：
- API 端点测试
- 数据库交互测试
- 控制器与服务的集成测试
- 多层架构集成测试

**测试示例**：
```java
@SpringBootTest

class SupplierControllerIntegrationTest {

```

#### 7.2.3 安全测试（Security Testing）

**测试框架**：Spring Security Test

**测试范围**：

- 认证机制测试

### 7.4 测试流程

1. **测试驱动开发（TDD）**：在编写代码之前编写测试用例
2. **持续集成（CI）**：在代码提交时自动运行测试
3. **测试隔离**：确保测试之间相互独立，不依赖执行顺序
4. **测试数据管理**：使用测试数据工厂或 fixtures 管理测试数据
5. **测试文档化**：为每个测试用例编写清晰的文档和注释
6. **定期重构测试**：保持测试代码的可维护性和可读性
7. **性能测试**：定期进行性能测试，确保系统性能满足要求

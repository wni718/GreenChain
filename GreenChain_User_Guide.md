# GreenChain 系统使用指南

## 1. 系统概述

GreenChain 是一个综合供应链可持续性平台，帮助企业跟踪、分析和优化其环境影响。系统提供全面的工具来监控碳排放量、管理供应商、跟踪运输路线，并生成详细的可持续性报告。

### 系统主要功能：
- 核心可持续性指标监控
- 碳排放趋势分析与可视化
- 供应商管理与认证状态跟踪
- 全球供应链地图可视化
- 运输跟踪与管理
- 可持续性报告生成

## 2. 用户类型及使用指南

GreenChain 系统支持四种用户角色，每种角色都有特定的权限和功能访问权限：

### 2.1 管理员 (ADMIN)

**角色概述**：管理员拥有系统的最高权限，可以管理所有功能和用户。

**主要功能**：
- 管理所有用户账户
- 管理所有供应商信息
- 查看和分析所有碳排放数据
- 生成全系统报告
- 配置系统设置

**使用流程**：
1. **登录系统**：访问登录页面，输入管理员凭据
2. **系统概览**：登录后进入首页，查看系统主要功能模块
3. **供应商管理**：在供应商管理页面添加、编辑或删除供应商信息
4. **数据分析**：在核心指标页面查看系统的关键可持续性指标
5. **报告生成**：使用报告生成功能创建详细的可持续性报告

**建议截图位置**：
- 登录页面：`/login` - 展示管理员登录界面
- 供应商管理页面：`/supplier-management` - 展示管理员如何管理供应商
- 核心指标页面：`/core-indicators` - 展示系统的关键可持续性指标

### 2.2 可持续发展经理 (SUSTAINABILITY_MANAGER)

**角色概述**：可持续发展经理负责监控和分析供应链的环境影响，制定可持续发展策略。

**主要功能**：
- 查看和分析碳排放数据
- 管理供应商信息
- 跟踪运输路线和环境影响
- 生成可持续性报告
- 接收智能推荐以优化环境绩效

**使用流程**：
1. **登录系统**：访问登录页面，输入可持续发展经理凭据
2. **数据分析**：在核心指标页面查看关键可持续性指标和历史分析
3. **供应商管理**：管理供应商信息，重点关注环境认证状态
4. **运输跟踪**：在运输跟踪页面监控运输活动及其环境影响
5. **报告生成**：创建详细的可持续性报告以评估进展

**建议截图位置**：
- 核心指标页面：`/core-indicators` - 展示历史分析和智能推荐部分
- 碳排放图表页面：`/carbon-emission-charts` - 展示排放趋势分析
- 报告生成页面：`/generate-report` - 展示报告生成界面

### 2.3 供应商 (SUPPLIER)

**角色概述**：供应商可以管理自己的信息，并查看与自身相关的环境影响数据。

**主要功能**：
- 查看和更新自身供应商信息
- 查看与自身相关的碳排放数据
- 查看自身的环境认证状态
- 查看与自身相关的运输活动

**使用流程**：
1. **登录系统**：访问登录页面，输入供应商凭据
2. **个人资料**：在个人资料页面查看和更新公司信息
3. **供应商信息**：查看自身的供应商详情，确保信息准确
4. **数据分析**：查看与自身相关的环境影响数据
5. **运输查看**：查看与自身相关的运输活动

**建议截图位置**：
- 个人资料页面：`/profile` - 展示供应商如何更新公司信息
- 供应商管理页面：`/supplier-management` - 展示供应商查看自身信息的界面
- 供应链地图页面：`/supplier-chain-map` - 展示供应商在全球供应链中的位置

### 2.4 查看者 (VIEWER)

**角色概述**：查看者可以查看系统中的数据和报告，但不能进行修改操作。

**主要功能**：
- 查看核心可持续性指标
- 查看碳排放趋势和分析
- 查看供应商信息
- 查看运输跟踪数据
- 查看生成的报告

**使用流程**：
1. **登录系统**：访问登录页面，输入查看者凭据
2. **系统概览**：在首页了解系统的主要功能
3. **数据分析**：查看核心指标和碳排放图表
4. **供应商查看**：浏览供应商信息和认证状态
5. **报告查看**：查看已生成的可持续性报告

**建议截图位置**：
- 首页：`/` - 展示系统的主要功能模块
- 核心指标页面：`/core-indicators` - 展示查看者可访问的指标数据
- 碳排放图表页面：`/carbon-emission-charts` - 展示排放趋势分析

## 3. 系统功能详细指南

### 3.1 核心指标 (Core Indicators)

**功能描述**：展示系统的关键可持续性指标，包括总碳排放量、避免的排放量、供应商认证状态等。

**使用方法**：
1. 登录系统后，从首页或导航菜单进入核心指标页面
2. 查看KPI卡片中的关键指标数据
3. 分析历史分析和智能推荐部分，了解潜在的减排机会
4. 查看各种图表以了解排放趋势和供应商分布

**建议截图**：
- KPI卡片部分：展示总碳排放量、避免的排放量等关键指标
- 历史分析部分：展示潜在节省和智能推荐
- 图表部分：展示排放趋势和供应商分布图表

### 3.2 供应商管理 (Supplier Management)

**功能描述**：管理供应商信息，包括添加、编辑、删除供应商，以及查看供应商的环境认证状态。

**使用方法**：
1. 从导航菜单进入供应商管理页面
2. 使用"New supplier"按钮添加新供应商
3. 使用"Edit"按钮编辑现有供应商信息
4. 使用"Delete"按钮删除供应商
5. 使用"Certified suppliers only"复选框筛选认证供应商

**建议截图**：
- 供应商列表：展示所有供应商的基本信息
- 添加/编辑供应商对话框：展示如何填写供应商详细信息
- 认证供应商筛选结果：展示筛选后的认证供应商列表

### 3.3 运输跟踪 (Shipment Tracking)

**功能描述**：跟踪和管理供应链运输活动，包括运输路线、碳排放等信息。

**使用方法**：
1. 从导航菜单进入运输跟踪页面
2. 查看运输列表，了解运输状态和环境影响
3. 添加新的运输记录
4. 编辑或删除现有运输记录
5. 分析运输的碳排放数据

**建议截图**：
- 运输列表：展示所有运输记录的基本信息
- 添加/编辑运输对话框：展示如何填写运输详细信息
- 运输详情：展示单个运输的详细信息和碳排放数据

### 3.4 供应链地图 (Supplier Chain Map)

**功能描述**：通过交互式全球地图展示供应商位置和运输路线，直观了解供应链的地理分布。

**使用方法**：
1. 从导航菜单进入供应链地图页面
2. 查看全球地图上的供应商位置标记
3. 查看运输路线的可视化展示
4. 点击标记查看供应商或运输的详细信息
5. 使用地图控件缩放和移动地图

**建议截图**：
- 全球供应链地图：展示供应商和运输路线的全局视图
- 区域放大视图：展示特定区域的供应商和运输路线
- 供应商详情弹窗：展示点击供应商标记后显示的详细信息

### 3.5 碳排放图表 (Carbon Emission Charts)

**功能描述**：通过各种图表可视化碳排放数据，帮助用户分析排放趋势和模式。

**使用方法**：
1. 从导航菜单进入碳排放图表页面
2. 查看不同类型的碳排放图表
3. 分析排放趋势和模式
4. 比较不同运输模式的碳排放
5. 了解碳排放的时间分布

**建议截图**：

### 3.6 报告生成 (Generate Report)

**功能描述**：生成详细的可持续性报告，包括碳排放数据、供应商认证状态、运输活动等信息。

**使用方法**：
1. 从导航菜单进入报告生成页面
2. 选择报告类型和时间范围

**建议截图**：

## 5. 最佳实践与建议

### 5.1 数据管理

- **定期更新供应商信息**：确保供应商数据的准确性和完整性
- **及时记录运输活动**：实时跟踪运输活动，确保数据的时效性
- **验证环境认证状态**：定期验证供应商的环境认证状态
- **分析历史数据**：利用历史分析功能识别改进机会

### 5.2 性能优化

- **使用筛选功能**：在处理大量数据时使用筛选功能提高性能
- **定期清理数据**：删除不需要的旧数据，保持系统性能
- **优化查询**：合理使用系统的查询功能，避免过度查询
- **使用批量操作**：在处理多个项目时使用批量操作功能

### 5.3 安全最佳实践

- **使用强密码**：为所有用户账户设置强密码
- **定期更新密码**：定期更改用户密码，提高账户安全性
- **限制访问权限**：根据用户角色设置适当的访问权限
- **保护敏感数据**：避免在公共场合展示敏感的环境数据

## 6. 故障排除

### 6.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 无法登录系统 | 用户名或密码错误 | 检查用户名和密码，确保正确输入 |
| 数据加载缓慢 | 网络连接问题或系统负载高 | 检查网络连接，稍后再试 |
| 无法添加供应商 | 权限不足或必填字段未填写 | 检查用户权限，确保填写所有必填字段 |
| 图表不显示数据 | 数据为空或加载失败 | 检查数据源，确保有数据可显示 |
| 报告生成失败 | 数据量过大或系统错误 | 减少报告范围，或联系系统管理员 |
